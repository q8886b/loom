# 004 — 分层重设计：目的与思想

## 分层结构

四层 = 认知深度递进：原文 → 一份材料 → 多份材料 → 元

- **第一层（原文捕获）**：是一张卡（`layer=L1, type=source`，008 §10），全文 markdown 存入 `cards.content`，原始路径记在 `source` 字段。**目的**：作为 L2 消化的素材 + 后续层按需精读的来源；与 L2/L3/L4 共用同一交互模型（search、read-cards、neighbors、graph、suggest-links 都覆盖 L1）。

- **第二层（精华理解）**：在第一层基础上消化这份材料的精华——提炼本质、找结构关系、形成整体论点。不跨材料。L2 的入口卡是"主题"卡，提供全局视野。**目的**：把材料变成可思考、可链接、可检索的认知单元——卡片网络的入口。

- **第三层（生成）**：基于第一二层的新产出。跨材料。举例（非穷尽）：主题研究、实践沉淀、跨域类比、综合判断、反例发现、教学输出。必须 link 至少一张 L2 卡——大模型不能凭空思考（008 §20；L1 source card 可作为原文证据补充，但不满足 L3 的最低门槛）。

- **第四层（元）**：思考方式本身——模式、哲学、洞察、方法论、世界观。不直接回答问题，但塑造你如何回答。两个方向：
  - **形成与演进**：动态的、多源的——人在交互中灌输的思想，或 AI 从第二层、第三层中抽象形成。这一层持续生成和演进。
  - **应用**：在消化新材料、生成新内容、解决新问题时，被检索和激活，作为思考框架——不被复制粘贴，被作为思考工具调用，应用过程也反哺这一层让它演进。
  
  **目的**：Loom 成为"有智慧的伙伴"的关键——不只是知识库，是认知架构。

## L1 → L2 的目的：两阶段 agent 分工

L2 agent 启动时读 L1 全文 markdown + L2 的主题卡，就有"已经读过一遍"的效果——记得全局结构，按需精读细节。模拟人脑第二遍读书的状态：记得结构和论点，不记得具体段落。

这个"已经读过一遍"的状态由两阶段子 agent 工程化实现——L2 处理一份材料（一本书或一份长材料）分两个阶段，**每个阶段是独立的子 agent**：

**第一阶段（Scout，通读 agent）**：
- 通读整本书（或所有已切分的 L1 markdown）→ 建立整本书的认知
- 回头建立每章主题卡（chapter-level 全局视野：整体论点、结构骨架）
- 模拟人脑"第一遍读书"——读完整本后回头建立每章主题
- 产出少（每章 1 张主题卡），可控

**第二阶段（Deep，精读 agent，每章一个并行）**：
- 启动时读该章 L1 全文 markdown + 该章主题卡（Scout 已建立）
- 主题卡已存在 = 实现"已经读过一遍"的状态
- 按需精读细节，产出该章其余 L2 卡
- 模拟人脑"第二遍读书"——记得结构和论点，按需精读细节
- 每章一个独立子 agent，context 互不污染，N 章并行可控

**为什么必须分两阶段**：把"通读 + 精读产出"的复合工作切成两次独立子 agent，每次 context 干净、产出可控。一次性读整本书 + 一次性产出所有卡片不可控——书越大越失控（一本 30 章的书一次性产出 300 张卡片做不到）。

**与 Adler 阅读层次的对应**：Scout = 检视阅读（抓结构、检视整体）；Deep = 分析阅读（理解内容、按需深入）。

## type 系统

8 种 type：

- **概念**：把特殊归入普遍。回答"这是什么"，建立认知的基本单元——包括定义、边界、分类归属、分型。所有推理的基础。
- **结构**：回答"它由什么组成 / 在哪里 / 怎么排列"。描述对象的空间、时间、度量、层级组织——静态的、无因果的排列。
- **机制**：回答"它如何发生 / 为什么变化"。揭示因果链、过程、反馈循环、条件触发——有时间方向的动力学。
- **案例**：具体发生过什么。提供经验材料，是判断的依据，模式抽象的原材料。
- **判断**：基于材料形成结论、做评价、做决策。两个维度——认识根据（为什么相信）和行动根据（为什么这样做）。
- **反思**：检查判断/模式的适用条件、例外、反例、冲突、误用、修正方向。三个层次——记录例外、完善定理、提出新定理（质疑框架本身）。**必须锚定一张已存在的判断/模式卡**——反思的本质是"对某个判断/模式的修正"，没有"被反思的对象"反思无意义。概念/结构/机制/案例不能单独支撑反思——它们是材料，不是判断。反思是元认知操作，对象必须是认知产物（判断/模式）。
- **模式**：从多个案例/判断中抽象出可迁移的稳定结构。跨域迁移的载体，类比推理的桥梁。成熟后可称原理。
- **主题**：对一份材料（或一组卡）的高层组织理解。L2 的入口卡，提供全局视野。**由 Scout 阶段建立**（不是 Deep 阶段的产出）——Deep agent 启动时主题卡已存在，是其"已读过一遍"状态的前提。

### L2 卡的 link 规则（设计澄清）

**L2 阶段（DIGEST）的 link 规则**（008 §1）：

1. **必选：link 主题卡**——非主题 L2 卡必须 link 对应主题卡（整批校验强制）
2. **可选：link 本材料内其他 L2 / L1 source card**——为了表达材料内部结构（同 domain:book 前缀内）
3. **反思卡额外锚定判断/模式卡**——由 type 系统硬约束保证

**禁止**：跨材料、跨领域、L4 的 link——这些是 **L3 THINK 阶段**的事。

**校验分层**：
- write-draft 内联：只检查"不跨材料/不跨领域/不 link L4"
- SubagentStop 整批：检查"非主题 L2 必须 link 主题卡"
- Deep 前置拦截：topic_card 存在性由 write-draft 校验 plan.topic_card

设计意图：DIGEST 专注消化单份材料（L1 → L2），本材料内 link 表达内部结构；跨材料/跨域网络化由 L3 THINK 阶段主动构建。

## type × layer 的约束

| type | L1 | L2 | L3 | L4 |
|---|---|---|---|---|
| source | ✓（专属）| × | × | × |
| 概念 | × | ✓ | ✓ | × |
| 结构 | × | ✓ | ✓ | × |
| 机制 | × | ✓ | ✓ | × |
| 案例 | × | ✓ | ✓ | × |
| 判断 | × | ✓ | ✓ | ✓（哲学级）|
| 反思 | × | ✓ | ✓ | ✓（对元原则）|
| 模式 | × | 少见 | ✓ | ✓（核心）|
| 主题 | × | ✓（入口）| ✓ | × |

补充约束：

- **type 体系（008 §13 方案 B）**：8 种认知 type + `source`。`source` 只能用于 `layer=L1`，不是认知 type——它是 L1 原文卡的 card type。
- L4 没有"元概念"——元层内容通过模式或判断表达，避免造抽象词
- L3 必须至少 link 一张 L2 卡（008 §20；L1 可补充但不满足门槛）——大模型不能凭空思考
- 比较通过 link + 卡内说明表达，不作为独立 type

## 设计的理论依据

- **Adler《如何阅读一本书》**——阅读有 4 个层次（基础 / 检视 / 分析 / 主题），递进式。检视抓结构，分析理解内容，主题跨书比较。对应 L1（检视）/ L2（分析）/ L3（主题）。

- **叔本华《论阅读与书籍》**——重要的书读两遍。两个原因：结构上，只有知道结局才能真正理解开头；视角上，第二遍心境不同，看到的东西不同。对应 L2 在 L1 基础上"再读一遍"。

- **诠释学循环（Schleiermacher / Dilthey）**——整体通过部分理解，部分通过整体理解，循环往复。理解是螺旋上升的。分层架构本身是循环的——L1 给 L2 全局视野，L2 反过来可能修订 L1 的理解。

- **Bacon《Of Studies》**——不是所有书都值得同等对待。4 种消化深度（浅尝 / 吞下 / 嚼 / 消化）。具体见下方"材料价值分级"。

- **Forte Progressive Summarization**——Just-in-Time vs Just-in-Case，不要前置整理，按需加工。触发条件是"再次使用"。L1/L2 是前置（消化材料时做），L3/L4 是按需（生成和元层被实际需要时触发）。

- **卢曼 Zettelkasten**——只做消化笔记，不做原文捕获。卡片是原子化的、用自己的话写的、密集链接的。L1 是 Loom 特有的——为 AI 消化设计的全局视野载体（卢曼本人不需要，人脑自带）。

## type 系统的理论来源

- **康德**——规定性判断力（概念）、时空直观（结构）、关系范畴（机制）、反思性判断力（反思、模式）、实践理性（判断）
- **叔本华四重根**——生成根据（机制）、存在根据（结构）、认识根据（判断）、行动根据（判断）
- **皮亚杰**——同化顺应（案例、判断、反思、模式）

## 材料价值分级（Bacon）

材料的质量、信息密度各异——不是所有材料都值得同等深度消化。按 Bacon 的 4 级：浅尝 / 吞下 / 嚼 / 消化。

分级有两个维度：

1. **跑到哪层**：L1 必做（即使暂时无法理解、当下无作用，也要保留原文，未来可能突然用到）；L2/L3/L4 按价值——高价值完整跑，中价值 L1 → L2（可能部分 L3），低价值只 L1 按需再 L2。

2. **每层处理密度**：相同层级下，不同材料的卡片产出密度不同。有些材料很多页才产出一张卡（叙事书、案例集）；有些每页多张卡（教材、方法论）。密度由材料的认知结构决定，不由字符数决定。

呼应 Forte——L1 是 just-in-case（保存），L2/L3/L4 是 just-in-time（按需深入）。

## 使用流程与效果

### 三个使用场景

**1. 消化（L1 → L2）**：新材料入网
- 触发：用户给新材料（书、文章、视频等）
- 流程：
  1. L1 切分（主 agent：把原始材料切成可独立消化的章节 markdown）
  2. Bacon 分级（主 agent：判断材料价值，决定 layer）
  3. **Scout 阶段**：一个子 agent 通读整本书 → 建立每章主题卡
  4. **Deep 阶段**：N 个并行子 agent，每个处理一章 → 产出该章其余 L2 卡
- L3L4 不在此时建（除非材料本身触发明显的生成/模式）
- **DIGEST 子 agent 完全 L4-blind**：不读 L4 索引、不 link L4 卡、专注消化材料；L4 连接是后续 THINK 阶段的事。发现新模式涌现可 propose-l4（不需要读已有 L4）

**2. 深入思考 / 反刍 / 跨领域（L3，可能 L4）**：
- 触发：多种——
  - 刚消化完一本书，把新卡接入已有网络（关联、对比、综合、反例）
  - 用户提出研究方向（"研究安全边际的跨流派应用"）
  - 反思某个已有判断
- 流程：反复查询+关联+思考，构建 L3 生成卡（综合、跨域、反例）和可能的 L4 新模式

**3. 具体问题和实践（用网络）**：
- 触发：问问题、做决策、复盘经历
- 流程：同反复过程。默认只回答；有新增、可复用、可支撑的认知单元时，可提示沉淀。确认后才写 L3 实践卡 / 反思卡。

### 反复过程（场景 2、3 共享）

不是一次性查询，是螺旋式深入：查询 → 关联 → 思考 → 再查询 → 循环，直到足够

### 如何读卡片网络（四种检索交替）

1. **关键词（FTS5）**：抓字面
2. **向量（embedding）**：抓语义相似
3. **结构性回忆（browse / children / siblings）**：抓主题骨架
4. **关系骨架翻译 + LLM 结构等同性判断**：抓 L4 模式

### L4 卡的特征

L4 卡（尤其是模式卡）不是抽象的、空洞的概念，而是**有血肉填充的**——包含核心模式、跨域实例、适用边界、反例误用、关联应用、修正演进等。类似毛选的矛盾论、实践论——是一整套思考体系，不是一句抽象原则。

不同模式卡的血肉形态不同（取决于模式本身的性质），通过 link 连接形成"模式 + 血肉"的思考体系。

**本体与网络的边界**：L4 卡本体不能空心化，至少要自足表达核心结构命题、成熟度、基本适用边界；跨域实例、反例、关联应用、修正演进等更丰富的血肉，可以通过 link 到 L2/L3/L4 卡形成网络来承载。L4 的完整形态是"本体命题 + link 血肉网络"，不是 30 字索引卡，也不是把所有实例都塞进单卡。

### L4 怎么激活

**核心定位**：L4 卡不是"被检索的模式库"，是 **LLM 思考时的扩展记忆 + 血肉来源 + 约束边界**。

LLM 通过预训练已经内化了大量模式（远比 Loom 的卡片库丰富）。Loom 不需要"激活"LLM 已有的模式，只需要：

1. **扩展 LLM 的思考方向**——LLM 启动读 L4 索引，知道 Loom 沉淀过哪些方向，思考时不遗漏
2. **提供具体"血肉"**——LLM 内化的是抽象骨架，Loom 的 L4 卡有跨域实例、适用边界、反例、反思等具体血肉
3. **约束 LLM 不跑偏**——L4 卡是已被验证的思考方向，把 LLM 拴在用户的真实认知轨迹上
4. **沉淀领域/用户特异性模式**——LLM 不知道的、用户独有的模式（预训练没见过的）
5. **可追溯记录**——用户过去思考的沉淀，下次直接调用

**L4 索引约定（content 第一段）**：

每张 L4 卡的 content 第一段是"核心结构命题 + 简短描述"——一句话讲清这个模式的本质结构。LLM 启动时读所有 L4 卡的 (title, content 第一段)，按需读完整 content。

**应用流程**：

```
1. LLM 启动读 L4 索引（title + content 第一段）
   → 知道 Loom 里有什么思考方向

2. LLM 思考问题时，主动用上相关的 Loom L4 卡
   → 不是被动检索，是主动调用

3. 用 Loom 卡的"血肉"（完整 content）补充自己抽象的内化
   → LLM 内化 + Loom 血肉 = 完整思考

4. 思考过程沉淀新洞察 → 可能更新 L4 卡
   → Loom 持续演化
```

**核心机制转变**：从"检索-匹配"到"思考-激活"。检索是被思考驱动的（按需深挖），不是思考被检索驱动。

**为什么这避开了"想得到 + 搜得到"的难题**：

- **想得到**：LLM 启动读索引，知道 Loom 有什么，不会"完全想不到"
- **搜得到**：不用精准搜——LLM 主动选一个相关的，读完整 content 自我修正。即使选错，下一次有反馈学习

**本质**：Loom 不是"被检索的知识库"，是"激活 LLM 思考能力的脚手架"。

## L4 模式的成熟度标注

L4 模式有成熟度差异，应用时不一样：

- **探索期**：新形成，还在验证中。应用时需要更多反思、反例检查
- **熟练期（第一性）**：反复使用、跨域验证、稳定可靠。激活优先级高

实现：约定 L4 卡 content 第一段开头标注 `[探索期]` 或 `[熟练期]`，不单独加字段。和"核心结构命题"写在一起。

## 卡片活跃度自动维护

所有卡片（包括 L1 的 markdown 文件）自动维护 2 个指标：

- `search_count`——被 search 命中的次数（出现在搜索结果中）
- `use_count`——被实际使用的次数（get / link / 激活等任何实际调用）

两者互补：
- search_count 高但 use_count 低 → 搜到了但没用，可能检索精准度问题或卡片质量不行
- use_count 高但 search_count 低 → 用得多但搜不到，可能靠其他方式（link、直接 ID）调用

实现：store.py / tools.py 在每次操作时自动记录。不做复杂分析，只累积 raw data。

后续可基于这两个指标做召回策略 / 沉默卡提醒 / 学习闭环。

## 卡片来源与 tag

卡片可区分认知主导来源：默认 `origin=ai`，表示由 AI 消化、总结、生成或整理形成；只有人在明确主动沉淀自己的想法、判断、原则时，才标记为 `origin=human`。`origin` 不表示最近编辑者，也不改变入库校验强度。

tag 是人显式维护的横切索引值，用于展示、筛选和统计；它不是 `type` / `layer` / `link` 的替代品，也不参与材料消化判断。AI 不主动根据内容给卡片打 tag；只有人的明确要求才添加或移除 tag。tag 不参与 embedding / FTS 排序，不改变卡片的认知结构。

## 4 层之间的依赖关系

- **L1 → L2 → L3 垂直依赖链**：
  - L2 依赖 L1（消化需要原文，L2 卡 source 字段指向 L1 source card）
  - L3 依赖 L1L2（生成需要素材 + 理解，L3 卡必须 link 至少一张 L2 卡——L1 可补充但不满足门槛，008 §20）

- **L4 跨层独立**：L4 不依赖具体材料，是元层思考方式。但 L4 的"血肉"（关联判断、案例）通过 link 横向连接到 L1L2L3。**跨材料 ≠ 跨领域**（008 §11）——单领域内可迁移的稳定结构默认沉淀为 L3；只有跨领域反复成立、能脱离具体领域成为思考方式本身的结构，才有资格进入 L4。L4 合格门槛是至少 link 到 2 个不同领域的 L2/L3 卡（L1 source card 可 link 但不计门槛，008 §19）。

### L4 血肉 = 本体 + link 网络（008 §11）

L4 卡（尤其是模式卡）的"血肉"不是塞进单卡——是**本体命题 + link 网络**的复合体：

- **本体**：自足表达核心结构命题、成熟度、基本适用边界（第一段承担索引功能，整卡不能空心化）
- **link 网络**：跨域实例、适用边界、反例、关联应用——分布在 L4 link 到的 L2/L3/L4 卡里
- L4 可以 link L1（作为原文证据），但跨域锚定门槛只统计 L2/L3

**embedding 是辅助，link 是显性真相**（008 §11）：embedding 用于 `suggest-links` 辅助构建 link 网络，但最终关联以显性 link 表达。两套机制不并存——link 是 truth，embedding 是 building tool。

### L4 激活：强制读入，不强制使用（008 §11）

L4 激活分两步：

1. **强制读入 context**：THINK/USE 启动时必须 `orient`，L4 索引进入 agent 视野。这是 harness 的职责——确保 agent 有信息可用，不靠 agent 自觉记得去读。
2. **不强制使用**：读入之后用不用、怎么用，是 agent 思考的自然结果。不设"每次 THINK 必须引用/link 至少 N 张 L4"这种硬约束——强制使用会扭曲思考（agent 合规式调用，为凑引用而引用，不是真用）。

Harness 对"是否用上"的观测手段：
- trace 记录所有 read-cards 调用
- 长期统计某次思考全程零 L4 引用 → WARN（不是失败），提醒优化 skill
- WARN 是信号，不是惩罚——可能是 L4 索引质量不够，也可能是这次思考确实不需要 L4

### 四项语义判据（008 §11）

语义层（SubagentStop 计算层通过后 block 回子 agent 做的自检）的四项判据，权威位置在 `skills/_loom_core.md` 与各模式 skill：

| 判据 | 推导来源 |
|---|---|
| type_match | type 系统（每种 type 回答什么核心问题）|
| single_unit | 独立认知单元 = 一句问句概括 |
| genuine_digest | 真消化非抄录原文 |
| self_contained | 不看 source 能否独立读懂 |

## L4 模式的演化机制

- **人工标注**：用户决定 L4 卡从 [探索期] 升级到 [熟练期]
- **大模型提醒**：基于**正向信号**（use_count 高 + 被反思修正次数多 + 跨域验证），提醒"这张卡可能该升级"——通过 `loom l4-upgrade-candidates` 按需调用，不自动触发
- **不自动升级**：决定权在用户
- **L4 演进必须经过 proposal / human review**：agent 不能自动创建或入库 L4 卡——走 `propose-l4` 写 staging，主 agent 汇报用户审核，用户批准后将 proposal status 改为 approved，再调 `loom-admin commit-l4` 入库

**L4 激活的入口**：不是裸 `l4_index.md` 文件，而是 `orient`（立体目录 = namespace 全貌 + L4 全量含核心命题摘要）。THINK/USE 启动时强制 `orient` 读入 context（§L4 激活段）。

## 卡片基本结构总览

**L2/L3/L4 卡的字段**：

- `id`：namespace + 卢曼 ID
- `title`：短标题
- `type`：8 种之一
- `content`：消化内容。第一段是索引——核心结构命题 + [探索期/熟练期] 标注（仅 L4）
- `links`：双向图邻接边
- `source`：描述性来源
- `origin`：认知主导来源，默认 `ai`；明确人工沉淀时为 `human`
- `tags`：人显式维护的横切索引值列表，默认空
- `use_count`：累计使用次数（自动维护）
- `search_count`：被 search 命中次数（自动维护）

**L1 的特殊性**（008 §10）：

- L1 是统一卡片体系中的一张卡（`layer=L1, type=source`），与 L2/L3/L4 共用同一交互模型
- L1 source card id：`<领域>:<书>:src:<单元ID>`（如 `llm:harness:src:08`）
- `content` 保存 markdown 全文（方案 C）；`source` 字段保留原始 markdown 路径
- 自动维护 use_count / search_count（与 L2/L3/L4 一致）
- 文件系统上的 markdown 是 L1 卡的全文来源/镜像，不是交互身份本身
