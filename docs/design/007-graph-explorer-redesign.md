# 007 - 图谱探索器重设计

## 目标

用"可浏览的结构"替代当前的扁平网格 + 局部聚焦。让用户在树形骨架中漫游，同时通过 link 飞线发现跨域关联。

## 技术选型

- 渲染层：**Vue Flow**（@vue-flow/core）
- 节点为真 Vue 组件，边为自定义 SVG path
- 布局由应用层计算坐标，Vue Flow 只做渲染 + 交互基建（pan/zoom/选择/拖拽）

## 整体布局

三栏结构保留：

```
┌──────────┬──────────────────────────────┬────────────┐
│ 左侧栏    │         画布（Vue Flow）       │  右侧面板   │
│ 域列表    │                              │  卡片详情   │
│ 快捷入口  │                              │  浏览历史   │
└──────────┴──────────────────────────────┴────────────┘
```

### 左侧栏（简化）

- 域列表：所有 namespace，标注卡片数，点击切换画布到该域的树
- 快捷入口：最近访问的卡片（5-10 条）、可选的收藏/书签
- 不再展示树形大纲（那是画布的职责）

### 右侧面板（保持不变）

- 选中卡片的完整内容：标题、type、layer、content、links 列表
- 浏览历史：前进/后退/下拉跳转
- 点击 links 列表中的卡片 → 画布联动跳转

## 画布：两种布局模式

### 模式一：树视图（默认）

**布局**：水平树，根节点（主题卡）在左侧，children 逐层向右展开。

```
主题卡 ─── 01 概念卡
        │       ├── 01a 机制卡
        │       └── 01b 案例卡
        ├── 02 结构卡
        │       └── 02a 判断卡
        └── 03 概念卡
```

**渐进展开**：

- 默认只展示第一级 children（主题卡的直接子节点）
- 折叠的节点显示聚合 badge：子节点总数 + layer 分布色条
- Hover 节点时，右侧出现 chevron 按钮（▶）；点击展开该分支
- 展开/折叠有平滑动画（子节点滑入/滑出）
- 展开一个分支不影响其他已展开分支的位置（空间稳定性）

**语义缩放（视距自适应）**：

| 缩放级别 | 节点表现 |
|---------|---------|
| Zoom out（鸟瞰）| 色块：矩形填充 type 颜色，描边 layer 颜色，无文字。看整体结构/密度/分布 |
| Zoom in（阅读）| 双行卡片：200px 宽，第一行 type 标签 + layer 色条，第二行完整标题（最多两行约 24-28 字）|

缩放阈值根据节点可见面积自动切换，有渐变过渡（opacity fade）。

**Link 飞线**：

- 同域内 link：直接画贝塞尔曲线连接两个节点，颜色区别于树边（实线蓝色 vs 树边灰色虚线）
- 跨域 link：飞线从节点出发，指向画布边缘的**停靠区**

**边缘停靠区（Dock）**：

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│   [树视图主体]                           ┌────────┐ │
│                                          │ phil ×3│ │
│                                          │ fin  ×2│ │
│                                          │ llm  ×1│ │
│                                          └────────┘ │
│                                                     │
└─────────────────────────────────────────────────────┘
```

- 位置：画布右侧边缘（不随 pan/zoom 移动，固定在视口上）
- 内容：当前可见节点的跨域 link 目标，按目标域聚合
- 每个域条目显示：域名 + 色块 + link 数量
- 展开某个域：列出具体的目标卡片（标题 + ID 尾部）
- 点击目标卡 → 传送到该域的树，自动展开到目标卡位置并高亮

### 模式二：聚焦视图（双击进入）

**进入**：在树视图中双击任意节点，画布过渡到聚焦模式。

**布局**：中心节点居中，link 目标围绕排列，按域分组成簇。

```
            ┌─ phil 域 ─┐
            │  2b ── 2b1 │
            └────────────┘
                  │
    ┌─ fin 域 ─┐  │  ┌─ llm 域 ─┐
    │ 3a ── 3a1│──●──│ harness:12│
    └──────────┘     └───────────┘
                  │
            ┌─ gen 域 ─┐
            │  1b      │
            └──────────┘
```

- 中心节点（●）放大显示完整信息
- 每个域簇内的卡片保持卢曼父子关系（小型水平树片段）
- 超过一跳的卡 → 边缘停靠区
- 域簇之间有足够间距，视觉分组明确（底色区块或虚线包围框）

**退出**：点击"返回树视图"按钮，或点击停靠区某张卡传送过去（自动退出聚焦，进入目标域的树视图）。

**过渡动画**：进入/退出聚焦时，节点从树位置平滑滑动到辐射位置（反之亦然），而非瞬切。

## 节点视觉设计

### 折叠态节点（zoom in 可见时）

```
┌─────────────────────────────────────┐
│ ● 概念  ━━━━━ L3                    │  ← type 色点 + layer 色条
│ 康德先验感性论的时空直觉              │  ← 标题（完整或两行折行）
└─────────────────────────────────────┘
         ↓12 ▸                           ← 聚合 badge + hover 才出现的 chevron
```

- 宽度：200px 固定
- 高度：48-56px（根据标题行数）
- 左侧：type 对应的色点
- 上方/右上：layer 对应的细色条或描边
- 主体：标题文本，最多两行
- 右下角 badge（有 children 时）："↓N"表示折叠了 N 个子节点
- Hover 时右侧浮现 ▸ chevron

### 折叠态节点（zoom out 鸟瞰时）

```
┌────┐
│ ██ │  ← 纯色块，type 填充 + layer 描边
└────┘
```

- 大小：约 24×16px
- 无文字，靠颜色辨识
- 聚合 badge 仍可见（缩小为小圆点 + 数字）

### 选中态

- 描边加粗变蓝（primary 色）
- 微弹放大（scale 1.02）
- 右侧面板同步显示详情

### 聚焦中心节点

- 放大约 1.5 倍
- 显示完整信息：标题 + type + layer + link 数量
- 金色描边/光晕

## 边的视觉设计

| 边类型 | 样式 | 说明 |
|-------|------|------|
| 卢曼父子 | 灰色虚线，1px | 树的骨架，低调不抢眼 |
| 同域 link | 蓝色实线，1.5-2px，贝塞尔曲线 | 域内关联，需要被发现 |
| 跨域 link（飞线）| 彩色渐变，2px，指向停靠区 | 颜色渐变=起点域色→目标域色 |

选中/hover 某节点时：
- 与该节点直接相连的边高亮加粗
- 其余边降低透明度

## 交互汇总

| 操作 | 效果 |
|------|------|
| 单击节点 | 选中，右侧面板显示详情 |
| Hover 节点 | 显示 chevron（有 children 时）；相关边高亮 |
| 点击 chevron | 展开/折叠该节点的子树 |
| 双击节点 | 进入聚焦模式 |
| 滚轮 | 缩放（语义缩放：远=色块，近=文字卡片）|
| 拖拽画布 | 平移 |
| 点击停靠区的域 | 展开该域的目标卡列表 |
| 点击停靠区的具体卡 | 传送：切换到目标域树视图，展开并高亮该卡 |
| 点击左侧域列表 | 切换画布到该域的树 |
| ⌘←/⌘→ | 选中历史 前进/后退 |

## 默认状态

- 打开页面：显示 `gen` 域的树视图
- 左侧域列表全部可见
- 树展开到第一级 children

## 性能策略

- Vue Flow HTML 节点在几百节点量级下表现良好
- 渐进展开天然限制了同时渲染的节点数（用户手动控制可见范围）
- Zoom out 时切换为简化色块（减少 DOM 复杂度）
- 超出视口的节点可使用 Vue Flow 内置的 viewport culling
- 如果某个域卡片极多（>500），首次加载只渲染到第一级，聚合 badge 标注总数
